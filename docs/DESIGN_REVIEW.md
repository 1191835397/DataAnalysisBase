## 1. 总体结论

当前 v0.2 设计方向可以达成构想中的核心目标：

- **全市场可见**：通过 AKShare 批量快照覆盖 A 股主要标的
- **本地可视化**：React + FastAPI + DuckDB 能支撑本地仪表盘
- **自动监管**：快照 diff + 规则引擎可实现异动告警
- **重点股深度**：FocusLayer 保留多源融合、对账、DeepSeek 研报

但要真正“日常可用”，必须补强以下能力：

1. 数据管道状态与数据质量
2. 告警可解释与去重
3. 高频快照存储与聚合策略
4. UI 对旧数据、缺失数据、失败任务的显式提示
5. 明确全市场浅分析与重点股深分析边界

这些优化已同步补充到 PRD、监管文档和 UI 文档中。

## 2. 关键优化点

### 2.1 从“能拉数据”升级为“可信数据管道”

原设计只说明 30 分钟拉一次全市场快照，但没有定义：

- 快照是否完整
- 同步任务是否成功
- UI 读的是不是半成品数据
- 重试会不会重复入库

已优化：

- 增加 `market_snapshot_runs`
- 增加 `fresh/stale/partial/failed/offline` 数据状态
- 明确 `(snapshot_time, security_id, source)` 幂等键
- UI 只读成功或部分成功的最新快照

### 2.2 从“告警列表”升级为“可解释监管事件”

原设计能产生告警，但告警可解释性不足。

已优化：

- 告警记录保存 `rule_id`、`rule_version`
- 保存触发值 `trigger_value` 与阈值 `threshold_value`
- 规则增加生命周期：版本、启停、冷却窗口、说明模板
- 增加去重策略，避免同一规则刷屏

### 2.3 从“全市场深分析”调整为“双层分析”

全市场 5000+ 股票不适合全部执行财报、RAG、LLM 分析，否则成本和数据源压力都会失控。

最终边界：

| 层级 | 分析深度 |
|------|----------|
| MarketLayer | 行情、行业、异动、基础估值、规则告警 |
| FocusLayer | 多源对账、财务、资金流、新闻、AI 研报 |

这个边界是项目能落地的关键。

### 2.4 从“页面列表”升级为“可运维界面”

原 UI 设计有页面结构，但对失败状态和旧数据处理不足。

已优化：

- 顶栏显示快照状态
- 所有 API 返回 `data_status`
- 增加 `/system/status`
- 增加 System Status 页面建议
- 股票列表明确服务端分页 + 聚合表查询

### 2.5 从“直接查大表”升级为“聚合表优先”

高频快照 90 天约 2160 万行。DuckDB 可以承载，但 UI 不应直接扫历史明细。

已优化：

- `latest_market_snapshot`
- `industry_snapshots`
- `market_overview_snapshots`
- `market_snapshots` 超期归档 Parquet

## 3. 当前设计仍需注意的风险

| 风险 | 说明 | 建议 |
|------|------|------|
| AKShare 源站变更 | 免费聚合源可能接口失效 | Provider 健康检查 + 降级状态 |
| 行业分类不稳定 | 不同数据源行业口径不一致 | 首期固定东财行业，后续再加申万 |
| 盘中 30 分钟不是实时 | 不能用于高频交易 | UI 明示快照周期 |
| 规则误报 | 简单阈值会产生噪音 | 通过冷却窗口、行业上下文、重点股自定义规则降低噪音 |
| LLM 成本 | 全市场不能逐只 AI 分析 | LLM 只用于 FocusLayer |

---

## 4. 是否能达到构想目的地

### 可以达到的目标

- 本地打开仪表盘，看到 A 股全市场概况
- 按行业查看强弱和成分股
- 监听全市场基础异动
- 对重点股更高频跟踪
- 对重点股做数据融合、对账、AI 分析
- 把数据记录到本地，支持复盘和分析

### 不建议追求的目标

- 秒级全市场实时行情
- 5000 只股票全量财报 + AI 研报
- AI 直接给买卖结论
- 免费数据源下实现机构级稳定性

## 5. 推荐实施策略

### 第一优先级：先完成“市场可见”

Phase A 必须先跑通：

```text
AKShare 全市场快照
  → DuckDB 入库
  → latest_market_snapshot
  → FastAPI
  → React 市场总览 / 股票列表 / 行业排行
```

### 第二优先级：再完成“变化可知”

Phase B 重点不是规则数量，而是闭环：

```text
相邻快照对比
  → 规则触发
  → 告警可解释
  → UI 可筛选
  → WebSocket 推送
```

### 第三优先级：最后加“重点股深度”

FocusLayer 需要在全市场底座稳定后再做：

```text
watchlist
  → 5min 深度同步
  → 多源对账
  → 个股详情
  → DeepSeek 研报
```

---

## 6. 评审结论

当前设计经过补强后，已经具备清晰落地路径。它不是传统“下载数据再分析”的脚本，而是一个本地化、可观测、可扩展的市场监管系统：

```text
全市场负责发现变化
重点股负责解释变化
AI 负责辅助理解，不负责编造结论
```

这条路径能达到当前构想的目的地，并且保留后续扩展港股、美股、RAG、推送和回测的空间。
