# 最终实现效果说明

> 关联：[ARCHITECTURE.md](./ARCHITECTURE.md) · [ROADMAP.md](./ROADMAP.md)

本文档描述 DataAnalysisBase **全部里程碑完成后**，用户能看到的具体结果与日常使用方式。

---

## 1. 产品定位（一句话）

> **输入一只自选代码 → 系统自动从多个数据源拉数、对账融合 → 给你一份「数字可信、差异可见、事件清楚、AI 解读有据」的研究报告；每天早上再给你一份自选组合简报，有异常主动告警。**

最终形态不是「预测涨跌的炒股 App」，而是**围绕自选标的的投资研究工作台**。

---

## 2. 日常使用方式

```text
早上 8:00（自动）
  → 自选股同步数据
  → 多源对账
  → 生成《每日简报》
  → 有异常推送到微信/邮件

白天（手动）
  → python cli.py research 600519.SH      # 深度研究一只
  → python cli.py ask "宁德和比亚迪估值对比"   # 对话提问
  → python cli.py overview 300750.SZ         # 快速看全景

有公告/重大新闻时（自动）
  → Monitor 告警："600519 存在 L3 财务差异" / "发布业绩预告"
```

核心体验：打开报告就知道**这只票现在什么情况、数据可不可信、最近有什么事件**，而不是 AI 一句「建议买入」。

├─ 估值（融合后）──────────────────────────────────────┤
│ PE(TTM): 23.9    PB: 8.2    近20日: -2.3%           │
├─ 多源状态 ───────────────────────────────────────────┤
│ 行情：✓ 一致 (akshare/tushare 差 0.03%)              │
│ 估值：⚠ PE 差 1.7% (L1)                              │
│ 财务：✓ 一致                                         │
├─ 数据新鲜度 ───────────────────────────────────────────┤
│ akshare: 2026-06-14 08:00  │  tushare: 2026-06-13   │

**你能得到**：当前估值、趋势、**数据是否可信**、最近有什么事。

### 3.2 多源对账报告（reconcile）

当 AKShare 和 Tushare 数字不一致时，单独生成对账报告：

```markdown
# 600519.SH 多源对账报告 | 2026-06-14

| 字段     | AKShare | Tushare | 差异   | 级别 | 融合值 |
|----------|---------|---------|--------|------|--------|
| 收盘价   | 1688.00 | 1687.50 | 0.03%  | L0   | 1688.00 |
| PE(TTM)  | 24.1    | 23.7    | 1.7%   | L1   | 23.9   |
| ROE      | 32.5%   | 32.5%   | 0      | L0   | 32.5%  |

## AI 差异解读
PE 差异可能来自 TTM 盈利窗口不同、市值取值时点不同...
建议：估值分析用融合值 23.9，报告中标注双源差异。
```

**你能得到**：知道两个源差多少、该信谁、为什么，而不是盲信单一 App。

### 3.3 AI 深度研报（research）

```markdown
# 600519.SH 贵州茅台 研究报告
> 生成：2026-06-14 | 数据截至：2026-06-13 | 置信度：0.8

## 摘要
近3年营收增速放缓至高个位数，盈利质量仍强，ROE 维持 30%+...

## 财务趋势
| 年份 | 营收(亿) | 净利润(亿) | 毛利率 | ROE  |
| 2023 | 1505     | 747        | 91.9%  | 34%  |
| 2024 | ...      | ...        | ...    | ...  |
（数据来自 canonical_financials，非 AI 估算）

## 估值与同业
| 公司     | PE   | ROE  |
| 贵州茅台 | 23.9 | 32%  |
| 五粮液   | 18.2 | 25%  |

## 风险因素
1. 行业政策与消费税预期
2. 增速放缓下的估值溢价

## 多源数据说明
PE 存在 L1 差异，已取融合值。

---
免责声明：仅供研究参考，不构成投资建议。
```

**你能得到**：以前需 4~8 小时整理的财报与对比，压缩到约 5 分钟，且数字有出处。

## 4. 对话式提问（ask）

```text
$ python cli.py ask "茅台和五粮液哪个盈利质量更好？"

Agent 内部：
  → 拉 600519.SH、000858.SZ 融合财务
  → 查对账问题
  → 算 ROE、毛利率、现金流
  → DeepSeek 对比解读

输出：
  从 ROE、现金流/净利润比、毛利率稳定性看：
  - 茅台 ROE 32% vs 五粮液 25%（数据：canonical_financials）
  - 茅台经营现金流/净利润比更高...
  结论倾向：茅台盈利质量更稳，但估值 PE 23.9 vs 18.2...
  （不构成买卖建议）
```

**你能得到**：像问研究员一样提问，回答带数据和来源。

## 5. 本地文件与数据落盘

运行一段时间后，本地目录大致如下：

```text
DataAnalysisBase/
├── data/
│   ├── duckdb/analytics.duckdb    # 所有融合后的数字（可 SQL 查询）
│   ├── chroma/                    # 公告/新闻向量库
│   └── reports/
│       ├── daily_20260614.md      # 每日简报
│       └── 600519.SH/
│           ├── overview_20260614.md
│           ├── reconcile_20260614.md
│           └── research_20260614.md
```

可直接 SQL 查询融合数据：

```sql
SELECT trade_date, close, lineage_json
FROM canonical_daily_bars
WHERE security_id = '600519.SH'
ORDER BY trade_date DESC LIMIT 5;
```

**你能得到**：不只阅读报告，还可自行查数、回测、导出分析。

## 6. 分阶段可见结果

| 阶段 | 可见结果 |
|------|----------|
| **M1**（第 1 周） | `sync` 后本地有 K 线；`reconcile` 出对账表；`overview` 看收盘价和差异 |
| **M2**（第 2 周） | overview 增加 PE/ROE/财务；双源财务对账 |
| **M3**（第 3 周） | **`research` AI 研报**、`ask` 对话问答 |
| **M4**（第 4 周） | **每日简报**、公告 RAG 问答、**微信/邮件告警** |
| **M5** | 基金净值、重仓股分析 |
| **M6** | 港股如 `00700.HK` 同样支持 overview/research |

### 里程碑与可用程度

| 版本 | 说明 |
|------|------|
| **第一个日常可用版本** | ≈ M3：sync、对账、AI 研报 |
| **完整形态** | ≈ M4：自动日报 + 监控 + 对话研究 |
| **M1 骨架预览** | `overview` + `reconcile` 两份 Markdown 报告 |

## 7. 会有 / 不会有

| 不会有 | 会有 |
|--------|------|
| 「明天涨停买这个」 | 「这只票财务/估值/事件如何，数据是否一致」 |
| 花哨 K 线网页（首期） | Markdown 报告 + CLI + 可选 API |
| 自动下单 | 研究辅助，决策由人完成 |
| 100% 准确的 AI 预测 | 可复现数字 + 带引用的 AI 解读 |

后期可选扩展：Streamlit 面板、FastAPI 前端；**首期交付核心是报告与数据**，不是炒股 App UI。

## 8. CLI 命令与产出对照

| 命令 | 产出 |
|------|------|
| `sync 600519.SH` | DuckDB 数据更新 + raw_snapshots |
| `reconcile 600519.SH` | `reports/600519.SH/reconcile_YYYYMMDD.md` |
| `overview 600519.SH` | 终端全景 + 可选 Markdown |
| `research 600519.SH` | `reports/600519.SH/research_YYYYMMDD.md` |
| `ask "..."` | 终端对话输出（可重定向保存） |
| `daily` | `reports/daily_YYYYMMDD.md` |
| `monitor run` | 告警日志 + 推送通知 |
