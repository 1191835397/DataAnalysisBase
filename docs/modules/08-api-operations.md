# api 模块使用说明

| 属性 | 值 |
|------|-----|
| 模块 | `api` |
| 最近更新 | `2026-06-22` |
| 关联设计 | [08-api.md](./08-api.md) |

## 1. 模块用途

把后端聚合数据和系统状态暴露为前端消费的 REST/WS 接口。

## 2. 运行方式

```bash
uvicorn dataanalysisbase.api.main:app --reload --port 8000
```

## 3. 配置项

| 配置 | 含义 | 默认值 | 风险 |
|------|------|--------|------|
| API 端口 | FastAPI 对外端口 | `8000` | 冲突会启动失败 |
| CORS | 前端访问控制 | 待定 | 配错会影响联调 |
| DuckDB 只读连接 | API 访问模式 | `true` | 误写会与同步冲突 |

## 4. 输入输出

### 输入

- 查询参数
- 分页和排序参数

### 输出

- `Envelope<T>`
- 统一错误结构

## 5. 常见问题排查

| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 前端 404 | 路由未挂载或 base path 错误 | 检查 `/api/v1` 路由注册 |
| 没有 `data_status` | `Envelope` 未统一注入 | 检查 `envelope.py` |
| 接口很慢 | 直接扫大表而非聚合表 | 检查 repository 查询路径 |

## 6. 扩展方式

- 新增接口先补 schema，再补 router，再补验证记录
- 避免在 router 中写业务计算逻辑

## 7. 变更注意事项

- 改返回结构会影响 frontend 类型和缓存逻辑
