# frontend 模块使用说明

| 属性 | 值 |
|------|-----|
| 模块 | `frontend` |
| 最近更新 | `2026-06-22` |
| 关联设计 | [09-frontend.md](./09-frontend.md) |

## 1. 模块用途

为用户提供本地 Web 仪表盘，展示市场总览、行业和股票列表。

## 2. 运行方式

```bash
npm run dev
```

## 3. 配置项

| 配置 | 含义 | 默认值 | 风险 |
|------|------|--------|------|
| Vite 端口 | 前端开发端口 | `5173` | 冲突会启动失败 |
| `/api` 代理 | 转发到后端 | `localhost:8000` | 配错会导致联调失败 |

## 4. 输入输出

### 输入

- API `Envelope` 数据
- 用户筛选、分页、排序操作

### 输出

- 页面渲染
- 顶栏状态
- 列表交互结果

## 5. 常见问题排查

| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 页面空白 | 路由或 API client 错误 | 检查控制台和网络请求 |
| 顶栏状态不显示 | `meta.data_status` 未传递 | 检查类型和组件 props |
| 股票列表卡顿 | 前端拿了过大数据量 | 检查服务端分页与表格实现 |

## 6. 扩展方式

- 新页面先补路由、类型、API client，再接组件
- 保持所有页面共享 `AppShell` 和状态组件

## 7. 变更注意事项

- 修改 API 类型时要同步更新 hooks、query key 和页面组件
