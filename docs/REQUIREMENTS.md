## 1. 产品愿景

构建一套 **本地可运行的 A 股全市场智能监管与分析平台**：

- 覆盖 **全 A 股**（沪深京 ~5000+ 只）的行情采集与变化监听
- 提供 **本地 Web 仪表盘**，直观展示市场、行业、个股与告警
- 支持 **行业分类** 浏览与板块级分析
- 对 **重点自选股票** 进行更高频跟踪、多源融合、深度分析与 AI 研报
- 从 **多个数据平台**（AKShare、Tushare 等）获取最新数据，本地记录、展示与分析

---

## 2. 用户与场景

| 角色 | 场景 |
|------|------|
| 个人投资者 | 每日开盘前后查看全市场概况、行业强弱、涨跌停分布 |
| 个人研究者 | 按行业筛选标的，发现异动股，加入重点跟踪 |
| 开发者本人 | 本地部署、扩展数据源、对重点股做多源对账与 AI 分析 |

为了避免系统停留在“架构完整但难以日常使用”的状态，v0.2 以以下结果作为成功标准：

## 3. 功能需求

### 3.1 全市场行情（Must）

| ID | 需求 | 说明 |
|----|------|------|
| FR-M01 | 全 A 股列表 | 展示全部上市股票：代码、名称、行业、现价、涨跌幅、成交额等 |
| FR-M02 | 定时快照 | 盘中每 **30 分钟** 批量拉取全市场行情并入库 |
| FR-M03 | 市场总览 | 大盘指数、涨跌家数、涨停/跌停数、成交额等汇总 |
| FR-M04 | 日终日 K | 收盘后增量同步全市场日 K 线 |
| FR-M05 | 数据新鲜度 | UI 明示最后快照时间与数据源状态 |

### 3.2 行业分类（Must）

| ID | 需求 | 说明 |
|----|------|------|
| FR-I01 | 行业维度 | 按东财/申万等行业分类组织股票 |
| FR-I02 | 行业排行 | 行业涨跌幅、成交额排行与热力图 |
| FR-I03 | 行业成分 | 点击行业查看成分股列表，支持排序筛选 |
| FR-I04 | 行业异动 | 行业整体涨跌超阈时产生监管事件 |

### 3.3 自动化监管（Must）

| ID | 需求 | 说明 |
|----|------|------|
| FR-S01 | 变化检测 | 对比相邻快照，检测价格、成交量等异常变化 |
| FR-S02 | 规则引擎 | 可配置规则：涨停跌停、涨跌幅、放量、行业异动等 |
| FR-S03 | 告警流 | 全市场告警列表，支持级别/行业/时间筛选 |
| FR-S04 | 实时推送 | WebSocket 推送新告警（新快照处理完成后） |
| FR-S05 | 全市场覆盖 | 监管规则默认作用于全部股票，不限于自选 |
| FR-S06 | 告警解释 | 每条告警展示触发规则、指标值、阈值和快照时间 |
| FR-S07 | 告警去重 | 同一标的同一规则在短时间内去重，避免刷屏 |
| FR-S06 | 告警解释 | 每条告警展示触发规则、指标值、阈值和快照时间 |
| FR-S07 | 告警去重 | 同一标的同一规则在短时间内去重，避免刷屏 |

### 3.4 重点股跟踪（Must）

| ID | 需求 | 说明 |
|----|------|------|
| FR-F01 | 自选列表 | 用户配置 watchlist，独立于全市场列表 |
| FR-F02 | 高频同步 | 重点股盘中每 **5 分钟** 深度同步（行情、资金流、新闻等） |
| FR-F03 | 多源对账 | 重点股支持 AKShare + Tushare 融合与差异展示 |
| FR-F04 | 深度分析 | 重点股 K 线、估值、财务、对账报告 |
| FR-F05 | AI 研报 | DeepSeek 生成个股研究报告（数字来自本地库） |

### 3.5 数据平台接入（Must）

| ID | 需求 | 说明 |
|----|------|------|
| FR-D01 | Provider 抽象 | 统一接口，可插拔 AKShare、Tushare |

| FR-D04 | 融合血缘 | 重点股融合值附带 lineage |
| FR-D05 | 数据质量 | 入库时记录完整率、字段缺失、源状态、快照状态 |
| FR-D06 | 数据归档 | 高频快照支持保留期与 Parquet 归档策略 |
| FR-D05 | 数据质量 | 入库时记录完整率、字段缺失、源状态、快照状态 |
| FR-D06 | 数据归档 | 高频快照支持保留期与 Parquet 归档策略 |

## 4. 非功能需求

| ID | 类别 | 要求 |
|----|------|------|
| NFR-01 | 部署 | 单机本地运行，Windows 友好 |
| NFR-02 | 规模 | 支持 ~5000 股票 × 48 快照/天 |
| NFR-03 | 延迟 | 全市场非 tick 级；30min 快照为设计基准 |
| NFR-04 | 可用性 | 单数据源失败不崩溃，UI 显示降级状态 |
| NFR-05 | 合规 | 免责声明；数据仅供研究 |
| NFR-06 | 安全 | API Key 不入库、不提交 git |
| NFR-07 | 查询性能 | 常用页面首屏查询目标 < 2s，股票列表服务端分页 |
| NFR-08 | 可观测性 | 同步任务、数据源、告警引擎均有状态表和最近错误 |
| NFR-09 | 幂等性 | 同一快照时间重复执行不会产生重复数据或重复告警 |

## 5. 数据更新策略（已确认）

| 范围 | 频率 | 内容 |
|------|------|------|
| 全 A 股 | 30 分钟 | 批量现货快照（`stock_zh_a_spot_em`） |
| 重点股 | 5 分钟 | 行情、资金流、新闻标题、估值 |
| 全 A 股 | 日终 1 次 | 日 K、行业分类、证券主数据刷新 |
| 重点股财务 | 周 / 按需 | 财报与财务指标（不全市场拉取） |

## 6. 明确不做

- AI 直接预测涨停、买卖信号
- 全市场 5000 只财报自动全量同步
- 交易所 Level-2 毫秒级行情
- 自动下单交易
- 替代 Wind / Choice 机构终端

## 7.1 运行模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `dev` | 本地开发，手动触发同步，热更新前后端 | 开发调试 |
| `local-live` | 本机常驻，按交易时段自动快照和告警 | 日常使用 |
| `replay` | 使用历史快照重放监管规则 | 规则调试与回归测试 |
| `offline` | 不访问外部数据源，仅查看本地已入库数据 | 网络异常或复盘 |
